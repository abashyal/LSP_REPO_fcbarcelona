# Question 2 – Design Explanation: Template Method Pattern

The Template Method pattern is implemented in the abstract `Report` class, which defines the fixed workflow in the `final` method `generateReport()`. This method calls, in order: `loadData()`, `formatHeader()`, `formatBody()`, and `formatFooter()`—the skeleton of the algorithm is locked and cannot be overridden. The variable steps (`loadData`, `formatHeader`, `formatBody`, `formatFooter`) are declared `abstract` in `Report` and must be implemented by each concrete subclass. `StudentReport` and `CourseReport` each provide their own data and formatting without touching the overall workflow. The `Driver` stores both report types in a `List<Report>` and loops through them, calling `generateReport()` polymorphically—demonstrating that the superclass controls the algorithm while subclasses supply only the details.
